# ssh_connect

A small Bash utility to select an SSH target with `dialog` and connect quickly.

The script keeps UI and logic separate:
- `dialog` is used only for selection.
- Connection logic is handled in dedicated Bash functions.

## Features

- Interactive server selection via `dialog --menu`
- SSH execution after selection
- Optional `--dry-run` mode to print the command instead of executing
- Optional `--config <path>` to use a custom TOML file
- Optional `--init` to create a complete example config file
- Safe abort if `--init` target already exists

## Requirements

- Bash (macOS default Bash 3.2 is supported)
- `dialog`
- `yq` (used to parse TOML)
- `ssh`

## Usage

```bash
bash ./ssh_connect.sh
```

### Options

```text
--dry-run           Show SSH command after selection, do not execute it
--config <path>     Use a custom TOML config file (default: ssh_connect_server.toml)
--init              Create an example config file and exit
-h, --help          Show help
```

Examples:

```bash
bash ./ssh_connect.sh --dry-run
bash ./ssh_connect.sh --config ./my_servers.toml
bash ./ssh_connect.sh --init --config ./my_servers.toml
```

## Configuration Format (TOML)

The script currently loads entries from `[server.<id>]` sections.

Required fields per server:
- `ip`
- `user`

Optional fields per server:
- `name` (display label in menu)
- `certificate` (path to SSH key file)

Example:

```toml
[server.app_prod]
name = "App Production"
ip = "203.0.113.10"
user = "deploy"
certificate = "/Users/youruser/.ssh/app_prod.pem"
```

### Notes about groups

The example config generated by `--init` also includes group-related fields:
- `[group.<id>]`
- `servers = ["server.x", ...]`
- `group_certificate`
- `[server.<id>.override]`

These fields are included as a template for future extension. The current runtime selection reads only `[server.*]` entries.

## What each function does

- `fail(message)`
  - Prints an error to stderr and exits with non-zero status.

- `usage()`
  - Prints command help and available options.

- `parse_args("$@")`
  - Parses CLI flags (`--dry-run`, `--config`, `--init`, `--help`).

- `init_config_file()`
  - Creates an example TOML config file with all supported template fields.
  - Aborts if the target file already exists.

- `check_dependencies()`
  - Verifies `yq` and `dialog` are installed.

- `load_servers()`
  - Reads server entries from the TOML file via `yq`.
  - Fills internal arrays (`SERVER_KEYS`, `SERVER_NAMES`, `SERVER_IPS`, `SERVER_USERS`, `SERVER_CERTS`).
  - Skips incomplete server entries and fails if no valid entries exist.

- `select_server_index()`
  - Builds menu items and shows the `dialog` selection UI.
  - Returns selected array index or non-zero on cancel.

- `run_selected_server(index)`
  - Builds the SSH command for the selected server.
  - In `--dry-run`, prints the exact command and exits.
  - Otherwise runs `ssh` using `exec`.

- `main("$@")`
  - Orchestrates the full flow:
    1. Parse args
    2. Optional init mode
    3. Validate config file and dependencies
    4. Load servers
    5. Open selection menu
    6. Execute (or print) SSH command

## Exit behavior

- Cancel in dialog -> prints `Cancelled.` and exits cleanly.
- Missing config / missing tools / invalid data -> exits with a clear error message.

## Project files

- `ssh_connect.sh` - main script
- `ssh_connect_server.toml` - default config file
